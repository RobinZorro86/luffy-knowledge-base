# 任务分级与优先级制度

> 版本：v1.0 | 2026-04-14
> 适用：Luffy + Zorro-Ops/Research/Edu
> 核心：不是所有任务都同等重要，资源永远稀缺，优先级是生命线

---

## 优先级定义

### P0 — 最高优先级（阻塞性）

**定义**：系统挂了、核心流程中断、数据丢失风险

**处理原则**：
- 立即处理，不等
- 遇到 P0，Luffy 亲自上手，不下放
- 可以牺牲 P1/P2/P3 的任务来处理 P0

**示例**：
- Camofox 端口挂了 → 立即诊断 + 修复
- X Cookies 失效、推文抓取全面失败 → 立即处理
- Cron 任务连续失败超过 3 次 → 立即排查
- 磁盘空间 > 95% → 立即清理

### P1 — 高优先级（当日必须完成）

**定义**：影响当日核心工作流，Robin 明确要求的任务

**处理原则**：
- 当日完成
- 优先于 P2/P3
- 失败必须上报

**示例**：
- Robin 发的推文入库 → 当日完成
- 新闻快报 cron 失败 → 当日排查
- Zorro 子 Agent 阻塞 2 小时未解决 → 上报

### P2 — 中优先级（本周内完成）

**定义**：优化类、改进类任务，不影响现有流程

**处理原则**：
- 本周完成
- P1 任务优先
- 可以推迟到下周

**示例**：
- Wiki 整理（大批量 frontmatter 补全）
- SOP 文档优化
- 新 cron 任务测试
- 知识库健康检查

### P3 — 低优先级（有空再做）

**定义**：探索性、实验性任务，做了更好，不做也不影响

**处理原则**：
- 有空才做
- 任何 P1/P2 任务出现，P3 自动让位
- 可以无限期推迟

**示例**：
- 尝试新的推文抓取工具
- 研究新的 Agent 架构
- 非紧急的 wiki 结构重组

## 任务分级流程

### 判断优先级的问题清单

遇到新任务时，Luffy 必须先问：

```
1. 如果这个不做，系统/流程会挂吗？
   → YES → P0

2. Robin 今天明确说了要做吗？
   → YES → P1

3. 这个任务本周内做和下周做，差别大吗？
   → YES → P2
   → NO → P3
```

### 优先级调整

- **升级**：遇到新信息导致优先级变化 → 立即调整并通知 Robin
- **降级**：任务完成后发现价值不如预期 → 降级处理

## 任务状态定义

### 执行状态

| 状态 | 含义 | 说明 |
|------|------|------|
| `pending` | 待处理 | 已接收，还没开始 |
| `in_progress` | 进行中 | 正在做 |
| `blocked` | 阻塞 | 遇到问题卡住了 |
| `waiting_on_robin` | 等待 Robin | 需要 Robin 决策才能继续 |
| `done` | 已完成 | 交付完成 |
| `cancelled` | 已取消 | 明确取消，不做了 |

### 质量评级（针对已完成任务）

| 评级 | 含义 | 后续处理 |
|------|------|---------|
| `✅ pass` | 全部达标 | 直接交付 |
| `⚠️ warn` | 部分达标 | 修复非核心问题后交付 |
| `❌ fail` | 不达标 | 打回重做 |

## 阻塞上报制度

### 什么情况必须上报

1. P0 任务阻塞超过 30 分钟
2. P1 任务阻塞超过 2 小时
3. 预期外的系统问题（不是任务本身的问题）
4. 需要 Robin 做决策才能继续

### 上报格式

```
## 🔴 阻塞上报 — YYYY-MM-DD HH:MM

**任务**：[任务描述]
**优先级**：P0 / P1
**阻塞原因**：
- [原因1]
- [原因2]
**已尝试**：
- [尝试1]
- [尝试2]
**建议**：
- [方案A]
- [方案B]
**是否可降级处理**：是/否
```

## Feature Flag 思维

> 核心：每个任务/功能都是"可开关的"，差的结果应该被关闭而不是放行

### 任务开关

| 开关 | 含义 | 行动 |
|------|------|------|
| `enabled` | 正常执行 | 按计划执行 |
| `paused` | 暂停 | 不做，等通知 |
| `bypass` | 绕过 | 跳过，直接交付 |
| `archived` | 归档 | 不再做 |

### 判断是否 bypass

```
任务完成了，但质量评级是 ❌ fail：
→ 能不能绕过？（Robin 能接受这个结果吗？）
  → YES → bypass 交付，标注问题
  → NO → 打回重做
```
