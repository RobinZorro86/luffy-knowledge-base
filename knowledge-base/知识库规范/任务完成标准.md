# 任务完成标准

> 版本：v1.0 | 2026-04-14
> 适用：Luffy + Zorro-Ops/Research/Edu 所有任务
> 核心原则：任务做完不等于做好，"做好"必须有明确标准

---

## 新闻快报任务

### 触发方式
- Cron 自动执行（每日 17:00）
- Robin 手动触发

### 完成标准（必须全部满足）

| # | 标准 | 检查方式 |
|---|------|---------|
| 1 | 每板块 ≥2 条新闻 | 人工抽查 |
| 2 | 每条含：标题 + 来源 + 日期 + 2-3句摘要 | 格式检查 |
| 3 | 日期必须是当天或昨天 | 时间戳验证 |
| 4 | 中文为主，英文术语附中文翻译 | 人工抽查 |
| 5 | 无事实性错误（人名/地名/数据） | 依赖信息源质量 |
| 6 | 按格式输出，不多解释 | 直接交付 |

### 质量评级

- **✅ 通过**：满足以上全部标准
- **⚠️ 降级**：缺1-2条新闻、格式略有偏差 → 标注"数据可能不全"交付
- **❌ 不通过**：时间错误（不是当天）、内容空洞（无实质信息）→ 重做

### 交付格式

```
## 📊 今日全球热点快报 — YYYY-MM-DD

### 🤖 AI与科技前沿
...

### 🌍 国际军事与地缘政治
...

---
资料来源：xxx / xxx / xxx
```

## 推文入库任务

### 触发方式
- Robin 发链接 + "入库/学习"
- Zorro-Research 主动发现值得入库的推文

### 完成标准（必须全部满足）

| # | 标准 | 检查方式 |
|---|------|---------|
| 1 | frontmatter 5字段：title/author/date/source/category | YAML 检查 |
| 2 | tags ≥ 3个 | YAML 检查 |
| 3 | 正文有 `> 核心观点 <` 引言块 | 内容检查 |
| 4 | 正文 ≥200 字（长推文/thread） | 字数统计 |
| 5 | 末尾有 `## 延伸参考` 关联已有 wiki | 内容检查 |
| 6 | 文件命名：`{年份}-{关键词}.md` | 文件名检查 |
| 7 | 推文 URL 可访问（抓取成功） | 执行验证 |
| 8 | `status: curated` | YAML 检查 |

### 质量评级

- **✅ 通过**：满足全部8条
- **⚠️ 降级**：缺1-2条非核心字段 → 交付但标注缺失项
- **❌ 不通过**：URL抓取失败 / 正文<50字 / 核心观点缺失 → 重做

### 目录结构

```
~/wiki/twitter/{作者名}/
└── {年份}-{推文关键词}.md
```

## 知识库整理任务（frontmatter 补全）

### 触发方式
- Zorro-Ops 例行整理
- Luffy 一次性批量任务

### 完成标准

| # | 标准 | 检查方式 |
|---|------|---------|
| 1 | 目标文件 frontmatter 5字段齐全 | YAML 检查 |
| 2 | 正文内容未修改（仅在头部插入） | diff 对比 |
| 3 | 完成后写整理报告 | 报告存在 |

### 整理报告模板

```markdown
## 知识库整理报告 — YYYY-MM-DD

| 文件 | 日期 | category | 说明 |
|------|------|----------|------|
| ... | ... | ... | ... |

处理说明：
- 未修改正文
- 来源优先从正文提取
- category 按规范匹配
```

## 系统监控任务（ops-log）

### 触发方式
- Zorro-Ops 每日 cron
- Luffy 手动触发

### 完成标准

| # | 标准 | 检查方式 |
|---|------|---------|
| 1 | 检查结果写入 `~/wiki/hermes/teams/ops-log.md` | 文件存在 |
| 2 | 每条记录含：时间戳、检查项、状态、说明 | 格式检查 |
| 3 | 异常项明确标注原因和建议操作 | 内容检查 |
| 4 | 正常项记录"✅ 正常" | 内容检查 |

### ops-log 格式

```markdown
## 系统自检报告 — YYYY-MM-DD HH:MM

### Camofox（端口9377）
- 状态：✅ 正常 / ❌ 异常
- 说明：...

### Docker
- 状态：✅ 正常 / ❌ 异常
- 说明：...

### Disk Space
- 状态：✅ 正常（XX%使用）/ ⚠️ 警告（>80%）/ ❌ 危机（>95%）
- 说明：...

### Token 消耗（本周）
- 状态：✅ 正常 / ⚠️ 注意
- 说明：...

---
下次自检：YYYY-MM-DD HH:MM
```

## Zorro 子 Agent 任务通用标准

### 完成标准

| # | 标准 |
|---|------|
| 1 | 任务目标明确达成（结果存在/输出符合格式） |
| 2 | 执行过程有记录（写到对应 memory 或 ops-log） |
| 3 | 遇到阻塞主动上报（不是默默失败） |
| 4 | 不确定的地方主动标注（不是猜着做） |

### 阻塞上报格式

```markdown
## 阻塞报告 — YYYY-MM-DD HH:MM

任务：[任务描述]
阻塞原因：[具体原因]
已尝试：[尝试过的方法]
建议：[下一步建议]
```

## Cron 任务完成标准

### 通用标准（所有 cron）

| # | 标准 |
|---|------|
| 1 | 输出内容符合 skill 指定的格式要求 |
| 2 | 日期是当天 |
| 3 | 错误时记录原因，不静默失败 |
| 4 | 执行结果记录到 ops-log（异常情况） |

### 技能特定格式

- **news-aggregator skill**：用 `## 📊` / `## 🤖` 格式
- **zorro-research**：按快报格式
- **zorro-ops**：按 ops-log 格式
